# Changelog

User-facing changes to PerfLint for Unity. This project follows [Semantic Versioning](https://semver.org/).

## [Unreleased]

## [1.1.0] — 2026-07-04

The "stuck Unity upgrade" release — turns a project that won't compile after a Unity 6 upgrade into a fixable checklist, and fixes most of it for you.

### Added
- **AI Migrate (Pro): one-click structural migrations, starting with RenderTargetHandle → RTHandle.** Some migrations can't be fixed by replacing one line — URP's removed `RenderTargetHandle` changes declarations, overrides, allocation and release across the whole file. AI Migrate rewrites the entire file following a curated migration playbook, probes your actual URP version by reflection to pick the right pass shape (compatibility mode vs. Render Graph — the model follows facts, not guesses), shows you exactly what changed (unchanged sections are folded), and only writes after you approve. If the rewrite doesn't compile, PerfLint feeds the compiler errors back to the AI and retries automatically (up to 2 rounds per click). Safety first: every round is compile-verified and **auto-rolls back on failure — your project is never left worse than before the click**. Privacy: unlike AI Fix (snippet-only), this sends the whole file to the AI provider — the panel states the exact line count and provider before every send, and nothing is sent until you click Generate. Current limits, stated honestly: files up to 500 lines, and you should still eyeball the visual result in your scene — a compile pass can't judge rendering. (This release ships several migration recipes — see the next entries.)
- **Any compile error is now a finding — and most are fixable.** PerfLint captures your project's C# compiler errors as they happen and turns them into per-file findings (Migration domain, Critical), so a project that won't compile after a Unity upgrade shows you exactly which scripts are blocked and why — even errors no curated rule knows about. Errors in package code get a separate summary pointing at the usual root cause (package version mismatch). Each broken script offers **AI Migrate (Pro)**: a whole-file rewrite guided by that file's exact error list plus a playbook of common upgrade breakages (FindObjectOfType family, WWW→UnityWebRequest, GetInstanceID→GetEntityId including the EntityId type ripple, Application.LoadLevel, and more), with the same compile-verify → auto-rollback → error-driven retry loop as all migrations. Note: fixing one file can surface follow-up errors in files that call into it — rescan after each fix; PerfLint reports these honestly rather than chaining rewrites automatically.
- **AI Migrate now repairs broken shaders too (Pro)**: on a "Shader fails to compile" finding (SHDR004), the Migration Assistant rewrites the file the compiler error actually points at — usually an included .hlsl, not the .shader shell — guided by your project's real URP version, the ShaderLibrary includes that actually exist on disk, and the shader's full error list. Verification is immediate and thorough: the shader is re-imported and its passes are actively re-compiled (catching lazy HLSL errors a plain re-import would miss); a failed rewrite rolls back automatically and feeds the fresh compiler errors back to the model (up to 2 retries per click). Hard guards refuse any rewrite that renames the shader or drops a material property — materials bind by those names. Same privacy contract as all AI Migrate actions: the panel states the exact file and line count before anything is sent.
- **"Why is everything magenta?" now has full coverage.** Two new checks join the existing pipeline-mismatch rule (MAT001), so each of the three common causes of pink materials gets its own precise diagnosis:
  - **Shader fails to compile (SHDR004)**: shaders whose source no longer compiles — typical after a Unity or render-pipeline upgrade breaks include paths, macros or syntax. The finding quotes the first compiler error (with line number) and the error count, and points you at the migration path. Reported from Unity's own recorded compile state, so it's deterministic.
  - **Material has a missing shader (MAT003)**: materials whose shader reference is None or points to a deleted/uninstalled shader. Previously these were silently skipped by the scan.
- **Migration check for `GetInstanceID` (MIG.GetInstanceID)**: newer Unity 6 releases deprecate `Object.GetInstanceID()` in favor of `GetEntityId()` — at error level it stops your project from compiling. PerfLint detects this precisely by asking your installed engine (reflection over the actual `[Obsolete]` attribute), so there are no false alarms on editors where the API is still current. Heads-up in the guidance: this is not a plain rename — the `EntityId`→`int` implicit conversion is obsolete there too, so the variables and dictionary keys receiving the id must change type along with it.
- **Migration check for URP's `RenderTargetHandle` (MIG.RenderTargetHandle)**: deprecated since Unity 2022.1 (URP 13) and removed in Unity 6, this type is one of the most common compile blockers when bringing older URP content (store assets, samples, custom render passes) to Unity 6. PerfLint now pinpoints every usage with line-level Locate and lays out the RTHandle migration path (allocate in `OnCameraSetup` via `RTHandles.Alloc` / `RenderingUtils.ReAllocateIfNeeded`, pass handles directly, release explicitly). Reported on Unity 2022.1+ only — on 2021 LTS the API is still current, so no noise there.
- **Mipmap Streaming advisor**: when a sizable pool of your scene textures could be served on demand, PerfLint quantifies it ("~X MB of textures could load only the mip levels the camera needs") and can enable Texture Streaming project-wide in one click — Quality Settings switch plus the per-texture Stream Mip Maps flags (Pro). Also flags the half-dead configuration where textures request streaming but the global switch is off.
- **Texture Streaming tuning deck** in the Runtime Profiler: live readouts (resident vs. desired vs. budget-capped texture memory, "streaming is currently saving ~X MB"), an over-budget warning when the Memory Budget is visibly costing quality, and all streaming parameters editable live in Play Mode — the mip debug view SRP doesn't ship. Unity reverts Play Mode changes to Quality Settings on exit, so the deck remembers what you tuned and offers a one-click apply back in edit mode (edit-mode changes persist directly). Every parameter belongs to the active quality level, which the deck displays so a runtime level switch never looks like lost tuning.
- **Static Batching memory bill (PERF.SBATCH001)**: static batching copies every batched instance's vertices into build-time Combined Meshes. PerfLint now estimates that hidden cost for your loaded scenes, names the top contributing meshes, and offers a one-click toggle-off (Pro) — presented as an informed frame-time ↔ memory trade-off, not a blind recommendation.
- **AssetBundle built-in duplication (ASSET.ABDUP002)**: built-in resources (default particle texture, built-in shaders, default font…) referenced by multiple bundles get silently copied into each — now detected with example referencing assets and a concrete replacement recipe.
- **Addressables compression check (ASSET.AACOMP001)**: local groups built with LZMA (slow full-decompress loads with zero download benefit) are flagged, with a one-click switch to LZ4 (Pro).
- **Bundle granularity check (ASSET.AAGRAN001)**: judged from your last real Addressables build output — flags fragmentation (many sub-512KB bundles: header overhead, OS file-handle limits) and oversized bundles (>50MB), with the 2–5MB mobile reference band.
- **Resources ↔ Addressables duplication (ASSET.AARES001)**: assets living in Resources folders that Addressables content also bakes copies of — the player always ships the Resources copy, so every dependent bundle adds an extra one. This is where the biggest offenders hide (a real project had one TMP font atlas duplicated 74× ≈ 1.16 GB of build size), and they're invisible to bundle-duplicate analysis because Resources assets aren't valid Addressable candidates. Report-only with concrete guidance (move out of Resources; TMP-specific notes when applicable).

### Changed
- **AI Migrate now spells out its credit cost.** On the hosted AI service, each request costs one credit — and because a whole-file migration can automatically retry after a failed compile, it may use a few. The panel now says so up front. (Bring-your-own-key users pay their own tokens and are never counted.)
- **Critical severity now means "this blocks your project."** Findings that stop compilation or guarantee broken rendering are now Critical (red) instead of Warning: removed APIs (`Application.LoadLevel`, `GUIText`/`GUITexture`, legacy particles), `RenderTargetHandle` on Unity 2023.1+/6 (error-level there; still Warning on 2022 where it merely warns), error-level `GetInstanceID`, and shaders that fail to compile. Deprecation warnings that don't block anything stay Warning — so a red badge in your scan now reliably means "fix this to get the project running." Unresolved assembly-definition references stay Warning accordingly: Unity skips them and compiles anyway (the Inspector greys them out), so they're usually migration leftovers to clean up — the finding now explains exactly when they do and don't matter.
- The shareable **HTML report** now carries a small, unobtrusive "Generated by PerfLint for Unity — perflint.dev" line in its footer. It's a plain link that only opens when clicked — the report still loads nothing and calls out to nothing on its own, and your scan is never uploaded.

### Fixed
- **Fixed a compile error on Unity 2021.3** introduced with the Texture Streaming tuning deck: two UI field types moved namespaces in Unity 2022.1, so the deck failed to compile on 2021 LTS. All supported versions (2021.3 through Unity 6) now compile and pass the full test suite.
- **Duplicate-packing findings are no longer hidden by your ignore list.** Ignored paths (Plugins/, Dependencies/…) are meant to silence import-settings advice on third-party assets — but a third-party asset duplicated into many bundles bloats *your* build, and fixing it never edits the third-party asset. Duplication findings (Addressables, AssetBundles, Resources overlap) now bypass ignore patterns; everything else still respects them. In a real project this was hiding the single biggest offender (a TMP font duplicated 74×).
- **Addressables duplicate detection (ASSET.AADUP001) no longer misses same-group duplication** — the biggest fish were slipping through. An asset baked into many bundles of ONE group (typical with one-bundle-per-scene groups; a real project had a font duplicated 74× ≈ 1.16 GB) previously wasn't reported at all, because duplicates were counted per group. Duplicates are now counted per bundle, matching the official Build Report. Findings also explain two special cases when they apply: same-group duplication (extraction still fixes it) and assets living in Resources folders (the Resources copy always ships — moving the asset out of Resources is the complete fix, typical for TMP fonts).
- **Finding details are now visible in the main panel.** Every finding row with details gets a ▸ caret — click the row to expand the full explanation (counts, guidance, trade-offs). Project-wide findings (no asset path) start expanded, since their substance lives entirely in the detail. Previously details only reached CSV/HTML exports and AI Explain, which left detail-heavy findings looking empty.
- **"Extract to shared group" (Addressables dedup) batch** now tells you what actually happened. After a rule-level "Extract all", a summary dialog reports how many assets were extracted, how many were skipped (read-only package assets can't be made Addressable), and — most importantly — the official duplicate-dependency count **before vs. after**, so you can see whether it actually reduced duplication. When it doesn't (common when the duplication is dominated by scene bundles that bake their own copies), it says so and points to the real fix instead of silently doing nothing. The batch also runs far faster (a single save instead of one per asset) with a progress bar, and no longer looks like "nothing happened" on success.
- **Runtime stutter spike (FPS003)** attribution is now reliable. The spike frame's call stack is captured the moment the freeze happens, so the finding pinpoints the script that triggered it (with Locate) instead of occasionally attributing the spike to an unrelated, quiet frame. When a spike occurs outside the sampling window and can't be captured, the finding now says so and points you to the Unity Profiler, rather than showing a misleading cause.

## [1.0.0] — 2026-06-30

First stable release — paid subscriptions are now live. What's included:

- **Local scan** across Performance, Assets, Migration, and Project Settings — one click, runs entirely on your machine.
- **Mobile-focused checks**: textures whose compression silently fell back to uncompressed, Multithreaded Rendering, Optimize Mesh Data, stereo clips that could ship as mono, and oversized fonts — each with a one-click fix where it's safe.
- **Project health score (0–100)** and a self-contained, shareable **HTML report**.
- **One-click & batch fixes** for safe, deterministic issues (Pro).
- **Merge duplicate assets** (Pro): keep one copy, redirect every reference across the project, then delete the rest — one click. Pick which copy to keep (defaults to the most-referenced one); guarded and previewed before it runs; commit to version control first.
- **Shader variants**: see how many variants each shader can produce, then **record** the ones your project actually uses and **warm them up** at startup (Pro) so they don't hitch on first use.
- **AI Fix & Explain** — zero-config out of the box, or bring your own API key. Every AI change is shown as a diff for review and is compile-verified with automatic rollback.
- **Local-first, zero telemetry** — your code and art assets are never uploaded.
